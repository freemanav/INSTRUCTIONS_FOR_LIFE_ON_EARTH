1. À PROPOS DU LIVRE ET DE L'AUTEUR
────────────────────────────────────────────────────

Ce livre a été écrit dans un but de transmission du savoir. L'auteur a choisi l'anonymat complet et ne recherche ni la gloire ni la reconnaissance. Ce livre n'est pas à vendre. Son code source est disponible sur GitHub, une plateforme reconnue comme l'une des plus stables et durables au monde.

GitHub a été choisi parce que :
- est une norme industrielle pour le stockage de code et de texte.
— Des millions de développeurs y stockent leurs projets depuis des décennies.
- La plateforme appartient à Microsoft, mais elle est basée sur le protocole ouvert Git.
- La probabilité que GitHub disparaisse dans 50 à 100 ans est extrêmement faible.

2. À PROPOS DES TRADUCTIONS DANS D'AUTRES LANGUES
────────────────────────────────────────────────────

Le russe est la langue originale. C'est la version la plus complète et la plus fidèle. Toutes les autres traductions s'appuient sur celle-ci.

Les 9 autres langues (anglais, grec, français, allemand, espagnol, chinois, arabe, hindi et portugais) sont traduites automatiquement à l'aide de Google Traduction intégré à Google Docs.

Ce que cela signifie pour le lecteur :
Le sens est transmis à 95-98%

L'auteur ne le cache pas. La traduction automatique est un choix délibéré.

3. QUELS FORMATS SONT INCLUS ET POURQUOI ILS LE SONT
────────────────────────────────────────────────────

Pour chaque langue, vous trouverez 3 (+1 en russe) formats principaux + police + instructions.

*** Formats et raisons :
──────────────────
1. Format DOCX
- Fichier Word original.
- À l'intérieur - mode « pas de saut de page ».
- Le livre se présente sous forme de texte continu, comme une page web.
- À partir de ce fichier, vous pouvez créer n'importe quel autre format, imprimer le livre, modifier la traduction.
- C'est le format le plus flexible.
──────────────────
2. Format HTML
- Page Web. S'ouvre dans n'importe quel navigateur.
- Également un texte continu, « fluvial », sans sauts de page.
- Idéal pour lire sur l'écran d'un téléphone, d'une tablette ou d'un ordinateur.
- Vous pouvez en faire un PDF (imprimer → enregistrer au format PDF).
──────────────────
3. Format TXT
- Texte brut sans mise en forme. Encodage UTF-8.
- C'est le format le plus éternel.
- S'ouvrira dans n'importe quel éditeur de texte, quel que soit le système d'exploitation.
Même si tout le reste reste fermé pendant 100 ans, TXT sera toujours ouvert. C'est votre assurance.
──────────────────
4. Format PDF (disponible uniquement dans la version russe du livre).
- C'est également l'un des formats les plus durables.
- Pratique pour l'impression.
IMPORTANT : Pour que le PDF s’affiche correctement, la police Verdana est requise. Si elle est manquante, le texte risque d’être déformé. C’est pourquoi elle est fournie séparément.
- En PDF, vous pouvez également définir le mode « continu » (sans sauts de page).
──────────────────

5. POLICE VERDANA (POURQUOI EST-ELLE UTILISÉE ?)
────────────────────────────────────────────────────

Le livre a été créé en utilisant la police Verdana, qui est la police standard de Windows présente sur la plupart des ordinateurs.

MAIS : d’ici 50 à 100 ans, cette police pourrait ne plus être disponible sur les nouveaux systèmes d’exploitation. Ou bien elle pourrait être remplacée par une autre.

Par conséquent, le fichier de police ORIGINAL Verdana.ttf a été inclus dans le dossier « Font_Verdana » avec les instructions d'installation.

Si vous voyez des caractères illisibles, des symboles étranges ou du texte qui ne s'affiche pas correctement lors de l'ouverture de fichiers HTML, DOCX ou TXT (et PDF), installez cette police.

COMMENT INSTALLER UNE POLICE :

WINDOWS :
1. Faites un clic droit sur le fichier Verdana.ttf
2. Sélectionnez « Installer »
3. Ou copiez-le dans le dossier C:\Windows\Fonts\

macOS :
1. Double-cliquez sur le fichier Verdana.ttf
2. Cliquez sur « Installer la police »
3. Ou copiez-le dans /Library/Fonts/ ou ~/Library/Fonts/

LINUX :
1. Copiez le fichier dans /usr/share/fonts/ ou ~/.fonts/
2. Exécutez la commande : fc-cache -fv
3. Redémarrez le programme dans lequel vous avez ouvert le livre.

Après l'installation, redémarrez votre navigateur ou Word. Tout devrait s'afficher correctement.

6. POURQUOI TOUS LES FICHIERS SONT-ILS DU TEXTE CONTINU (SANS SAUTS DE PAGE) ?
────────────────────────────────────────────────────

Il s'agit d'une décision consciente de l'auteur.

Un livre n'est pas un ouvrage imprimé aux pages fixes. C'est un flux continu de texte. Sur un téléphone, il occupera l'équivalent de 4 000 écrans. Sur un grand écran, 600. Sur une feuille A4, 800 pages. Sur une feuille A5, 1 600.

Et c'est normal.

L'auteur ne décide pas pour le lecteur où se termine la page. C'est le lecteur qui choisit l'appareil, la taille de la police, les marges et le format du papier.

Ainsi, le livre survivra à toutes les évolutions technologiques.

8. COMMENT IMPRIMER UN LIVRE
────────────────────────────────────────────────────

Tous les fichiers de cette archive sont en mode « sans sauts de page » (texte continu).
C’est un choix délibéré : le livre est conçu comme un texte « fluvial ».

Si vous souhaitez imprimer le livre :

Option 1 (plus facile) :
- Ouvrez le fichier HTML dans votre navigateur.
- Appuyez sur Ctrl+P (ou Cmd+P sur Mac).
- Dans les paramètres d'impression, sélectionnez le format de page, les marges, ainsi que les en-têtes et pieds de page souhaités.
- Imprimez-le.

Option 2 (plus flexible) :
- Ouvrez le fichier DOCX dans Word (ou LibreOffice, Google Docs).
Ajoutez vous-même des sauts de page là où vous le jugez nécessaire.
- Imprimez-le.

Ce livre n'est pas soumis à une mise en page rigide. Vous êtes maître de sa lecture et de son impression.
